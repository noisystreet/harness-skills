# Code Review References

## Guides

- [Google Engineering Practices: Code Review](https://google.github.io/eng-practices/review/) - reviewer and author expectations.
- [Conventional Comments](https://conventionalcomments.org/) - labeling feedback intent.
- Language and security skills in this repo for specialized checks.

## What To Learn

- Optimize for correctness, regressions, and readability of the change under review.
- Separate blocking issues from optional nits.
- Prefer small PRs with clear test plans.

## Caveats

- Team culture differs on nit density; match the repository's review norms.
