# Docs Style References

## Guides

- [Keep a Changelog](https://keepachangelog.com/) - user-facing change documentation.
- [Architecture Decision Records](https://adr.github.io/) - lightweight decision records.
- [C4 Model](https://c4model.com/) - diagrams for communicating architecture.
- [Semantic Versioning](https://semver.org/) - version meaning for release notes.
- [Google developer documentation style guidance](https://developers.google.com/style) - clarity and consistency heuristics.

## What To Learn

- Write for a specific reader and task, not for completeness theater.
- Keep one source of truth; link instead of copying rules.
- Put irreversible choices in ADRs, operational how-to in runbooks/README.

## Caveats

- Style guides are defaults; project tone and audience win when they conflict.
