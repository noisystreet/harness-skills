# Refactoring References

## Books And Guides

- `Refactoring` (Fowler) - behavior-preserving steps and smells.
- `Working Effectively with Legacy Code` - characterization tests and seams.
- [refactoring.com](https://refactoring.com/) - catalog of refactorings.

## What To Learn

- Keep tests green between small steps.
- Separate behavior changes from structural changes.
- Use strangler/expand-contract for risky replacements.

## Caveats

- Catalog names matter less than verifiable safety.
