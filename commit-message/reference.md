# Commit Message References

## Guides

- [Conventional Commits](https://www.conventionalcommits.org/) - type/scope/breaking format.
- [Angular commit guidelines](https://github.com/angular/angular/blob/main/CONTRIBUTING.md#commit) - influential conventional style.
- [Git documentation: commit messages](https://git-scm.com/book/en/v2/Distributed-Git-Contributing-to-a-Project#_commit_guidelines) - why/what clarity.

## What To Learn

- Explain why the change exists; keep subject imperative and specific.
- Use types consistently with the repository's history.
- Put breaking changes and migration notes where humans will see them.

## Caveats

- Not every repo enforces Conventional Commits; follow local history first.
