# Release References

## Guides

- [Semantic Versioning](https://semver.org/)
- [Keep a Changelog](https://keepachangelog.com/)
- [GitHub Releases docs](https://docs.github.com/en/repositories/releasing-projects-on-github)
- [SLSA](https://slsa.dev/) - provenance for release artifacts.

## What To Learn

- Cut changelog and tags together.
- Define rollout and rollback before pressing deploy.
- Separate code rollback from data migration rollback.

## Caveats

- SemVer for apps vs libraries may be interpreted differently; document local policy.
